  • Kmom01

    Hur känns det att komma igång med databaser, klienter och SQL - är det något du ser fram emot eller fasar inför?

    Jag ser fram emot det men fasar också inför det. Det är mycket att hålla koll på, och även en hel del saker som kan gå fel. Mitt största bekymmer just nu har varit att få igång docker, pga av en uppdatering som inte ville sig. Även sättet man tänker i SQL tar mig en stund att komma in i, eller ja backend över huvudtaget är nog nått som tar mig lite mer betänketid.

    Har du jobbat med databaser eller liknande tekniker tidigare?

    Jag har jobbat litegrann i MySQL sedan innan. Tror en utav min första full-stack appar va med Nodejs/Express, MySQL och React. Men det var också ett tag sedan jag gjorde den. Så känns bra att kunna gå igenom grunderna igen, och förhoppningsvis reda ut lite frågetecken på vägen.

    Berätta lite om ditt arbetet med SQL i guiden, hur gick det?

    Det gick ganska dåligt till en början, till jag hittade en bra rutin för att starta om databasen. När jag kom igång med det så var det lättare att kunna testa varje SQL statement och dubbelkolla att dem fungerade som dem borde. Det är mycket att hålla kolla på med MySQL, VS Code och mariadb terminalen, samt flödet som man behöver köra scripten för att det skall bli rätt. Väldigt lätt att missa nått om man inte ser vad man gör.

    Jämför SQL med andra sätt att programmera, relatera till några av de programmeringsspråk du kan.

    Tycker mest det påminner om python iallafall dem if statements där man kan skriva in (). Dock så känns SQL mer kompakt och med mindre funktionalitet. Logiken är nog det som påminner mest om andra sätt att programmera.
